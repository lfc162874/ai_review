# AI 复盘助手技术架构设计

## 1. 项目定位

AI 复盘助手是一款基于 AI Agent 的个人成长助手，通过文字、语音、对话等方式帮助用户完成周/月/年复盘，并沉淀用户成长轨迹。

核心能力：

- AI 对话式复盘
- 语音复盘
- 长期用户记忆
- 成长动态分享
- AI 成长建议

---

# 2. 整体架构

```
ai-review

├── ai-review-app
│   └── Vue3 + uni-app
│
├── ai-review-server
│   └── NestJS + TypeScript
│
└── ai-review-ai
    └── Pi Agent Runtime
```

# 4. 数据库架构

采用 PostgreSQL + pgvector 一体化方案。

原因：

- 支持业务数据
- 支持 AI 长期记忆
- 支持向量检索
- 降低系统复杂度

主要表：

```
user                 用户
review_record        复盘记录
conversation_message 对话记录
moment               成长动态
user_memory          用户长期记忆向量
```

---

# 5. 后端模块设计

## auth

负责：

- 登录
- JWT认证
- 用户授权

## user

负责：

- 用户资料
- 用户画像
- 成长等级

## review

核心业务模块。

支持：

- 日复盘
- 周复盘
- 月复盘
- 年度复盘
- 项目复盘

## conversation

AI 对话模块。

能力：

- 多轮对话
- 流式输出
- 历史上下文

## voice

语音复盘模块。

流程：

语音 -> ASR -> 文本 -> AI总结 -> 复盘记录

## memory

用户长期记忆模块。

通过 pgvector 保存：

- 用户习惯
- 用户目标
- 技术方向
- 成长经历

## moment

成长动态模块。

类似朋友圈：

- 发布动态
- AI总结卡片
- 点赞评论分享

# 6. AI架构

采用轻量 AI Agent 架构。

核心框架：

Pi Agent Runtime

不采用复杂工作流框架，保持轻量和可控。

结构：

```
ai-review-ai

├── agents
│   ├── review-agent
│   ├── coach-agent
│   └── growth-agent
│
├── tools
│
├── prompts
│
└── memory
```

---

# 7. Agent设计

## Review Agent

负责：

原始聊天内容 -> 复盘总结

输出：

- 完成事项
- 问题分析
- 成长点
- 下一步计划

## Coach Agent

负责：

- 成长建议
- 能力分析
- 目标规划

## Growth Agent

负责：

将复盘结果生成成长动态。
