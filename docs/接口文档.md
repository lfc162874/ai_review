# AI Review 后端接口文档

> Base URL: `http://localhost:3000/api`

## 通用说明

### 统一响应格式

所有接口返回统一 JSON 结构：

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

- `code = 0` 表示成功，非 0 为业务错误码
- 列表接口 `data` 中包含 `list`、`total`、`page`、`pageSize`

### 认证方式

需要认证的接口在请求头中携带 JWT Token：

```
Authorization: Bearer <accessToken>
```

### 分页参数

所有列表查询接口支持以下分页参数（Query 参数）：

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|---|---|---|---|---|
| page | number | 否 | 1 | 页码，从 1 开始 |
| pageSize | number | 否 | 20 | 每页条数，最大 100 |

### 枚举值

**复盘类型 ReviewType**

| 值 | 说明 |
|---|---|
| daily | 每日复盘 |
| weekly | 每周复盘 |
| monthly | 每月复盘 |
| yearly | 年度复盘 |
| project | 项目阶段复盘 |

**复盘状态 ReviewStatus**

| 值 | 说明 |
|---|---|
| draft | 草稿 |
| completed | 已完成 |
| generating | AI 正在生成总结 |
| generated | AI 总结已生成 |
| failed | AI 总结生成失败 |

### 错误码

| 范围 | 模块 |
|---|---|
| 10000-10999 | 通用错误 |
| 20000-20999 | 用户/认证 |
| 30000-30999 | 复盘 |
| 40000-40999 | 对话 |
| 50000-50999 | AI |

常用错误码：

| 错误码 | 说明 |
|---|---|
| 10000 | 未知错误 |
| 10001 | 参数校验失败 |
| 10002 | 未登录或登录已过期 |
| 10003 | 权限不足 |
| 10004 | 资源不存在 |
| 20001 | 用户不存在 |
| 20003 | 邮箱已被注册 |
| 20004 | 密码错误 |
| 20007 | 刷新 Token 无效或已过期 |
| 30001 | 复盘记录不存在 |
| 30002 | 复盘总结正在生成中 |
| 40001 | 对话会话不存在 |
| 40002 | 对话会话已关闭 |
| 50001 | AI 服务不可用 |
| 50002 | AI 生成超时 |

## 一、认证模块

### 1.1 用户注册

```
POST /api/auth/register
```

无需认证。使用邮箱和密码注册新用户，注册成功后返回 Token 对。

**请求参数（Body JSON）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| email | string | 是 | 用户邮箱，格式需合法 |
| nickname | string | 是 | 用户昵称，2-50 字符 |
| password | string | 是 | 登录密码，6-50 字符 |

**请求示例**

```json
{
  "email": "user@example.com",
  "nickname": "张三",
  "password": "123456"
}
```

**响应示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
  }
}
```

**错误码**

| 错误码 | 说明 |
|---|---|
| 20003 | 该邮箱已被注册 |

---

### 1.2 用户登录

```
POST /api/auth/login
```

无需认证。使用邮箱和密码登录，成功后返回 Token 对和用户信息。

**请求参数（Body JSON）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| email | string | 是 | 用户邮箱 |
| password | string | 是 | 登录密码 |

**请求示例**

```json
{
  "email": "user@example.com",
  "password": "123456"
}
```

**响应示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs...",
    "user": {
      "id": "a1b2c3d4-...",
      "email": "user@example.com",
      "nickname": "张三",
      "avatar": null
    }
  }
}
```

**错误码**

| 错误码 | 说明 |
|---|---|
| 20001 | 用户不存在 |
| 20004 | 密码错误 |

---


无需认证。使用刷新 Token 获取新的 Token 对。

**请求参数（Body JSON）**

```json
    "accessToken": "eyJhbGciOiJIUzI1NiIs...",
    "refreshToken": "eyJhbGciOiJIUzI1NiIs..."
  }

```

| 错误码 | 说明 |
|---|---|
| 20007 | 刷新 Token 无效或已过期 |

## 三、复盘模块

> 以下接口均需要 JWT 认证

### 3.1 创建复盘

```
POST /api/reviews
```

根据用户输入内容创建一条新的复盘记录。创建后状态为 `draft`，需单独调用 AI 生成接口触发总结。

**请求参数（Body JSON）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| type | string | 是 | 复盘类型，枚举值：`daily` / `weekly` / `monthly` / `yearly` / `project` |
| content | string | 是 | 复盘内容，纯文本，最多 10000 字符 |

**请求示例**

```json
{
  "type": "daily",
  "content": "今天完成了首页的开发，过程中遇到了一些布局问题，后来通过 flex 布局解决了。下午还修复了两个 bug，整体效率还不错。"
}
```

**响应示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "r1a2b3c4-...",
    "userId": "a1b2c3d4-...",
    "type": "daily",
    "content": "今天完成了首页的开发...",
    "summary": null,
    "status": "draft",
    "createdAt": "2025-01-15T10:00:00.000Z",
    "updatedAt": "2025-01-15T10:00:00.000Z"
  }
}
```

---

### 3.2 查询复盘列表

```
GET /api/reviews
```

分页查询当前用户的复盘记录，支持按类型和状态筛选，按创建时间倒序排列。

**请求参数（Query）**

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 20 |
| type | string | 否 | 复盘类型筛选：`daily` / `weekly` / `monthly` / `yearly` / `project` |
| status | string | 否 | 状态筛选：`draft` / `completed` / `generating` / `generated` / `failed` |

**请求示例**

```
GET /api/reviews?page=1&pageSize=10&type=daily
```

**响应示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "list": [
      {
        "id": "r1a2b3c4-...",
        "userId": "a1b2c3d4-...",
        "type": "daily",
        "content": "今天完成了首页的开发...",
        "summary": null,
        "status": "generated",
        "createdAt": "2025-01-15T10:00:00.000Z",
        "updatedAt": "2025-01-15T10:05:00.000Z"
      }
    ],
    "total": 1,
    "page": 1,
    "pageSize": 10
  }
}
```

> 以下接口均需要 JWT 认证

### 4.1 创建对话会话

```
POST /api/conversations
```

**请求参数（Body JSON）**

| 参数 | 类型 | 必填 | 说明 |

| title | string | 否 | 会话标题，最多 100 字符 |

**请求示例**

```json
{
  "title": "今日复盘对话"
}
```

**响应示例**

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": "c1a2b3c4-...",
    "userId": "a1b2c3d4-...",
    "reviewId": null,
    "title": "今日复盘对话",
    "status": "active",
    "createdAt": "2025-01-15T10:00:00.000Z",

        "role": "assistant",
```
